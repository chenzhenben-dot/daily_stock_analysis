# Claude Code 任务模板

> 本文件是仓库内 AI 协作任务的标准入口模板，便于把任务直接交给 Claude Code。
> 详细规则见 `AGENTS.md` 第 10 章"证据优先执行规范"。
> 创建任务前先把下方 13 个章节填齐；执行时逐项打勾并附证据。

## 0. 任务模板使用提示（执行时必读）

- **不得**把 fallback 成功当成主链路成功。
- **不得**把测试通过当成用户路径通过。
- 未验证项必须明确列出（按 `AGENTS.md` 10.1 节 5 种状态之一标注）。
- **未经用户授权**不得 commit / push / merge / 部署生产。
- 核心验收项未通过时，整体状态只能写"全部完成 / 部分完成 / 被阻塞 / 实现完成但尚未验收"中的合适项；只有所有适用的核心验收项已验证，且用户可见改动完成 L3，才能写"全部完成"。
- 执行时遇到 L3 验收被环境限制（无 LLM key / 无外部接口 / 维护窗口不允许），必须显式标注为"代码路径 L1 + L2 已验证，L3 因 <具体原因> 暂未验收"，整体状态为"实现完成但尚未验收"。
- 区分三种"修复"：实现已修复（允许使用，但必须附 L3 状态）、用户问题已修复（必须 L3 通过）、生产问题已修复（必须生产 L1 + L2 + L3 通过）。

```markdown

## 13.2 根因
- 一句话 + 链路证据

- 实现已修复：<是 / 否>
  - L3 状态：<已验收 / 未验收 / 受限，原因>
- 用户问题已修复：<是 / 否>（必须 L3 已验证）
- 生产问题已修复：<是 / 否>（必须生产 L1 + L2 + L3 已验证）

## 13.4 验收矩阵
```
